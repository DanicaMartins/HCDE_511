The US total looks almost back to Feb 2020, but sectors diverge sharply. Software Development is still 157 index points below its peak (February 2022). Physicians & Surgeons is closest to its own high (-10 pts). Highlighted bars are example sectors.

How we measure (index & data)
Indeed daily CSV → Monthly mean (total postings, SA) → 37 sectors → O*NET title match → Automation score · Rules in repo docs/DATA_RULES.md
How to read the index
  • 100 = posting volume on February 1, 2020.
  • 150 = 50% more postings than that day; 72 = 28% fewer.
  • Compare change over time and across sectors—not whether a value is “good.”
MetricMeaningNot
Posting index % of Feb 1, 2020 posting volume for that series (100 = same day). Raw job count, employment level, or wages.
Drawdown Current index minus sector peak index (negative = below peak). Permanent job loss count.
Automation score O*NET “degree of automation” matched via sector job titles, scaled 0–100. Claude automation vs augmentation (Anthropic metric).
AI releases OpenAI/Anthropic model publication months (Epoch-style registry). Proof that releases caused posting changes.

Limitations

What this does not show
  • Indeed does not publish raw posting counts in our dataset—only indexed change from Feb 2020.
  • 100 is a reference day, not a “healthy” labor market target.
  • Automation scores use keyword matching; some sectors fall back to a default when no occupation matches.
  • Correlation between AI releases and posting trends is not causation.
  • National totals near 100 in 2026 can hide sectors still far below their 2021–2022 peaks—see drawdown chart.
